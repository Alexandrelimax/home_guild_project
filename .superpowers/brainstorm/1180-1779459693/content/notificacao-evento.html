Como o player descobre que recebeu um evento?

O admin acabou de criar um evento e distribuir para os players. De que forma isso aparece para eles?

A

Sininho no header com badge de contagem

Um ícone de sino aparece no topo do app com um número vermelho (ex: "1"). Clicar abre um painel lateral ou dropdown listando os eventos pendentes. O número some quando o player abre o painel.

🔔1 Home Guild Dashboard · Histórico · Loja · Conquistas
B

Card de evento em destaque no Dashboard

Sem sininho. No dashboard do player, eventos novos aparecem numa seção separada "Eventos Ativos", acima das quests normais, com borda dourada/colorida. O destaque visual chama atenção sem precisar de notificação extra.

⚡ EVENTO ATIVO
🧹 Limpar o Banheiro · +150 XP · +30 Bits
C

Sininho + seção destacada no Dashboard

Combinação das duas: o sininho avisa que chegou um evento novo (some depois que o player abre o dashboard). No dashboard, eventos sempre ficam numa seção dedicada com visual diferente das quests comuns.

🔔1 Notificação no header → some ao ver o dashboard
⚡ Evento: Limpar o Banheiro · Seção separada das quests
